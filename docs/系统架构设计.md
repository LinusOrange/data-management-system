# 票据数据对账系统设计（基于“对账单 vs 入库单”的行级对照）

## 1. 目标

基于你提供的两类数据源：

- **来源 A：对账单**（包含“销售订单”“产品”等字段）
- **来源 B：入库单（进货明细表）**（包含“摘要”“型号”等字段）

系统要实现：

1. 按行组织两边数据并做可追溯对照。
2. 核心映射关系：
   - 对账单 `产品` ↔ 入库单 `型号`
   - 对账单 `销售订单` ↔ 入库单 `摘要`
3. 动态维护每条采购信息状态：
   - 是否完成对账
   - 是否完成开票
4. 提供看板统计：
   - 未对账条数
   - 未开票条数
   - 对账异常条数
   - 已闭环条数（已对账 + 已开票）

---

## 2. 数据组织原则（按行对照）

### 2.1 原始行永不覆盖

每次上传文件，先完整落地原始行，保留：

- 文件批次 ID
- Excel 行号
- 原始字段 JSON
- 解析状态

这样后续任何对账结论都能追溯到原始行。

### 2.2 统一标准行（Normalized Row）

把两边行数据映射到统一结构（核心字段）：

- `source_type`：`statement` / `inbound`
- `biz_date`：业务日期
- `order_ref`：订单引用（对账单=销售订单，入库单=摘要）
- `item_model`：型号/产品编码（对账单=产品，入库单=型号）
- `qty`：数量
- `amount_tax_incl`：含税金额（若存在）
- `customer_name` / `counterparty`
- `row_no`：源文件行号

后续对账全部基于标准行执行。

---

## 3. 行级对账规则（V1）

## 3.1 主匹配键（强匹配）

以以下组合键作为主匹配：

- `order_ref`（销售订单 ↔ 摘要）
- `item_model`（产品 ↔ 型号）

即：

```text
match_key = normalize(order_ref) + '|' + normalize(item_model)
```

> `normalize`：去空格、全角半角统一、大小写统一。

## 3.2 辅助校验字段

主键匹配后进行差异检查：

- 数量差异：`qty_diff = statement.qty - inbound.qty`
- 金额差异（若两边均有金额）：`amt_diff = statement.amount_tax_incl - inbound.amount_tax_incl`
- 日期偏差（可选）：是否超出容忍天数

## 3.3 匹配结果状态

- `matched_exact`：主键匹配且差异在阈值内
- `matched_with_diff`：主键匹配但数量/金额有差异
- `statement_only`：仅对账单存在
- `inbound_only`：仅入库单存在

并映射为业务状态：

- 对账状态 `reconciliation_status`
  - `pending`（未对账）
  - `done`（已对账）
  - `exception`（异常）

- 开票状态 `invoice_status`
  - `not_invoiced`
  - `partially_invoiced`
  - `invoiced`

状态判定建议：

- `matched_exact` -> `done`
- `matched_with_diff` / `statement_only` / `inbound_only` -> `exception`
- 未运行对账任务 -> `pending`

## 4. 系统模块设计

## 4.1 文件导入模块

- 上传 Excel
- 创建批次（含来源类型）
- 解析模板头部与明细区
- 写入 `raw_row`
- 映射写入 `normalized_row`

## 4.2 对账引擎

- 按批次或按时间窗拉取两边标准行
- 生成 `match_key`
- 进行 1:1、1:N、N:1 聚合匹配
- 输出 `reconciliation_pair` 与差异详情
- 回写采购状态

## 4.3 采购状态与开票维护

- 每个“业务行”（采购条目）独立状态
- 支持人工改状态与备注
- 所有变更写 `status_event` + `audit_log`

## 4.4 看板统计服务

按当前有效业务行实时汇总：

- 未对账：`reconciliation_status = pending`
- 对账异常：`reconciliation_status = exception`
- 未开票：`invoice_status in (not_invoiced, partially_invoiced)`
- 已闭环：`reconciliation_status = done AND invoice_status = invoiced`

## 5. 数据库结构（PostgreSQL DDL）

```sql
-- 1) 导入批次
CREATE TABLE import_batch (
  id                  BIGSERIAL PRIMARY KEY,
  source_type         VARCHAR(20) NOT NULL CHECK (source_type IN ('statement', 'inbound')),
  file_name           TEXT NOT NULL,
  uploaded_by         VARCHAR(64),
  uploaded_at         TIMESTAMP NOT NULL DEFAULT NOW(),
  parse_status        VARCHAR(20) NOT NULL DEFAULT 'pending',
  parse_error         TEXT
);

-- 2) 原始行（完整留痕）
CREATE TABLE raw_row (
  id                  BIGSERIAL PRIMARY KEY,
  batch_id            BIGINT NOT NULL REFERENCES import_batch(id),
  source_type         VARCHAR(20) NOT NULL,
  row_no              INT NOT NULL,
  raw_json            JSONB NOT NULL,
  created_at          TIMESTAMP NOT NULL DEFAULT NOW(),
  UNIQUE(batch_id, row_no)
);

-- 3) 标准化行（统一字段）
CREATE TABLE normalized_row (
  id                  BIGSERIAL PRIMARY KEY,
  batch_id            BIGINT NOT NULL REFERENCES import_batch(id),
  raw_row_id          BIGINT NOT NULL REFERENCES raw_row(id),
  source_type         VARCHAR(20) NOT NULL CHECK (source_type IN ('statement', 'inbound')),

  biz_date            DATE,
  order_ref           VARCHAR(100),    -- statement: 销售订单, inbound: 摘要
  item_model          VARCHAR(100),    -- statement: 产品, inbound: 型号

  qty                 NUMERIC(18,4),
  amount_tax_incl     NUMERIC(18,2),
  counterparty_name   VARCHAR(255),

  normalized_order_ref    VARCHAR(100),
  normalized_item_model   VARCHAR(100),
  match_key           VARCHAR(220),

  created_at          TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_norm_source_key ON normalized_row(source_type, match_key);
CREATE INDEX idx_norm_order_model ON normalized_row(normalized_order_ref, normalized_item_model);

-- 4) 对账任务
CREATE TABLE reconciliation_job (
  id                  BIGSERIAL PRIMARY KEY,
  statement_batch_id  BIGINT REFERENCES import_batch(id),
  inbound_batch_id    BIGINT REFERENCES import_batch(id),
  job_status          VARCHAR(20) NOT NULL DEFAULT 'running',
  started_at          TIMESTAMP NOT NULL DEFAULT NOW(),
  finished_at         TIMESTAMP,
  error_message       TEXT
);

-- 5) 行级对账结果（可支持1:1, 1:N, N:1）
CREATE TABLE reconciliation_pair (
  id                  BIGSERIAL PRIMARY KEY,
  job_id              BIGINT NOT NULL REFERENCES reconciliation_job(id),

  statement_row_ids   BIGINT[] NOT NULL DEFAULT '{}',
  inbound_row_ids     BIGINT[] NOT NULL DEFAULT '{}',

  match_key           VARCHAR(220),
  match_status        VARCHAR(30) NOT NULL CHECK (match_status IN (
                          'matched_exact',
                          'matched_with_diff',
                          'statement_only',
                          'inbound_only'
                        )),

  statement_qty_sum   NUMERIC(18,4),
  inbound_qty_sum     NUMERIC(18,4),
  qty_diff            NUMERIC(18,4),

  statement_amt_sum   NUMERIC(18,2),
  inbound_amt_sum     NUMERIC(18,2),
  amt_diff            NUMERIC(18,2),

  diff_detail         JSONB,
  created_at          TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_pair_job_status ON reconciliation_pair(job_id, match_status);
CREATE INDEX idx_pair_match_key ON reconciliation_pair(match_key);

-- 6) 业务采购条目（系统主视图，每条维护对账/开票状态）
CREATE TABLE purchase_item (
  id                      BIGSERIAL PRIMARY KEY,

  statement_row_id        BIGINT REFERENCES normalized_row(id),
  inbound_row_id          BIGINT REFERENCES normalized_row(id),

  order_ref               VARCHAR(100),
  item_model              VARCHAR(100),
  qty                     NUMERIC(18,4),
  amount_tax_incl         NUMERIC(18,2),

  reconciliation_status   VARCHAR(20) NOT NULL DEFAULT 'pending'
                          CHECK (reconciliation_status IN ('pending', 'done', 'exception')),
  invoice_status          VARCHAR(20) NOT NULL DEFAULT 'not_invoiced'
                          CHECK (invoice_status IN ('not_invoiced', 'partially_invoiced', 'invoiced')),

  latest_pair_id          BIGINT REFERENCES reconciliation_pair(id),
  remark                  TEXT,

  created_at              TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at              TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_item_recon_invoice ON purchase_item(reconciliation_status, invoice_status);
CREATE INDEX idx_item_order_model ON purchase_item(order_ref, item_model);

-- 7) 状态变更日志（开票/对账人工修正）
CREATE TABLE status_event (
  id                  BIGSERIAL PRIMARY KEY,
  purchase_item_id    BIGINT NOT NULL REFERENCES purchase_item(id),
  event_type          VARCHAR(30) NOT NULL,  -- reconciliation_update / invoice_update
  old_value           VARCHAR(30),
  new_value           VARCHAR(30),
  operator            VARCHAR(64),
  note                TEXT,
  created_at          TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 8) 通用审计日志
CREATE TABLE audit_log (
  id                  BIGSERIAL PRIMARY KEY,
  entity_type         VARCHAR(50) NOT NULL,
  entity_id           BIGINT NOT NULL,
  action              VARCHAR(50) NOT NULL,
  before_json         JSONB,
  after_json          JSONB,
  operator            VARCHAR(64),
  created_at          TIMESTAMP NOT NULL DEFAULT NOW()
);
```

## 6. 字段映射（按你给的两张图）

### 6.1 对账单 -> 标准行

- `创建日期` -> `biz_date`
- `销售订单` -> `order_ref`
- `产品` -> `item_model`
- `销售数量` -> `qty`
- `含税金额` -> `amount_tax_incl`
- `客户名称` -> `counterparty_name`

### 6.2 入库单 -> 标准行

- `单据日期` -> `biz_date`
- `摘要` -> `order_ref`
- `型号` -> `item_model`
- `数量` -> `qty`
- `折前金额`（或可用金额列）-> `amount_tax_incl`（若口径一致）
- `往来单位` -> `counterparty_name`

> 注意：金额口径（含税/未税）必须在导入模板层明确；不一致时只做数量对账，金额标记“不可比”。

---

## 7. 看板指标 SQL 示例

```sql
-- 未对账
SELECT COUNT(*) FROM purchase_item WHERE reconciliation_status = 'pending';

-- 对账异常
SELECT COUNT(*) FROM purchase_item WHERE reconciliation_status = 'exception';

-- 未开票
SELECT COUNT(*) FROM purchase_item
WHERE invoice_status IN ('not_invoiced', 'partially_invoiced');

-- 已闭环
SELECT COUNT(*) FROM purchase_item
WHERE reconciliation_status = 'done' AND invoice_status = 'invoiced';
```

## 8. 实施顺序建议（最短路径）

1. 完成两种 Excel 的解析器（按明细行抽取字段）。
2. 打通 `import_batch -> raw_row -> normalized_row`。
3. 实现主匹配键 `order_ref + item_model` 的对账任务。
4. 落地 `reconciliation_pair` 回写 `purchase_item` 状态。
5. 做看板接口与列表筛选。
6. 再加人工修正与开票维护页面。
